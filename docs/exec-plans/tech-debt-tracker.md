# Tech Debt Tracker

## 개요
에이전트와 사람이 공통으로 관리하는 기술 부채 목록입니다.
버그 수정, 리팩토링, 성능 개선 항목을 우선순위별로 관리합니다.
작업 완료 시 상태를 업데이트합니다.

---

## 우선순위 기준

| 레벨 | 기준 |
|------|------|
| 🔴 Critical | 게임 플레이 불가 or 크래시 유발 |
| 🟠 High | 주요 기능 비정상 작동 |
| 🟡 Medium | 성능 저하 or UX 불편 |
| 🟢 Low | 코드 품질 개선, 리팩토링 |

---

## 활성 부채 목록

현재 등록된 항목 없음 — 개발 진행 중 에이전트가 자동 등록합니다.

| - | - | - | - | - | - |

---

## 등록 형식

새 항목 발견 시 아래 형식으로 추가합니다.

```
| TD-001 | 🟠 High | 멀티볼 활성화 중 새 아이템 획득 시 볼 개수 초기화 버그 | Phase 1 | Implementor | 미해결 |
```

### 상태값
- `미해결` — 아직 수정 안 됨
- `진행 중` — 에이전트 작업 중
- `완료` — 수정 완료, 테스트 통과
- `보류` — 의도적으로 미룸 (이유 명시)

## 자동 등록 트리거 (에이전트용)

다음 상황에서 에이전트는 이 문서에 자동으로 항목을 추가합니다.

```
- flutter analyze 경고 발생 시
- 테스트 실패 후 즉시 수정 불가한 경우
- 리팩토링 필요하지만 현재 작업 범위 밖인 경우
- 성능 기준 미달 감지 시 (QUALITY_SCORE.md 참고)
- 스테이지 JSON 검증 실패 시
```
