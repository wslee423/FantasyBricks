# Core Beliefs — 게임 설계 철학

## 개요
이 문서는 개발 중 모든 의사결정의 기준이 되는 설계 원칙입니다.
새로운 기능 추가, 난이도 조정, 수익화 결정 시
반드시 이 원칙에 부합하는지 먼저 확인합니다.

---

## 1. 틈새 시간을 위한 게임

> "1판은 1~3분 안에 끝나야 한다"

- 스테이지 하나의 목표 클리어 시간: **90초 ~ 3분**
- 긴 로딩, 긴 연출, 강제 광고는 세션 흐름을 끊으므로 지양
- 언제든 중단 가능한 구조 (백그라운드 전환 시 자동 일시정지)
- 한 손으로 플레이 가능한 조작 복잡도 유지

**이 원칙에 위배되는 것들:**
- 스테이지 시작 전 5초 이상의 연출 강제 시청
- 클리어 조건이 복잡해서 1판이 10분 이상 걸리는 설계
- 멀티 터치가 필요한 조작 메카닉 추가

---

## 2. 아이템이 승패를 가른다

> "실력으로 버티고, 아이템으로 돌파한다"

- 초반 스테이지는 실력만으로 클리어 가능하게 설계
- 중반 이후부터 아이템 없이는 클리어가 어려운 난이도 커브
- 아이템은 전략적 선택이 아니라 **생존 수단**에 가까운 포지션
- 아이템 드롭 타이밍과 위치가 플레이의 핵심 긴장감을 만든다

**이 원칙에 위배되는 것들:**
- 아이템이 있어도 없어도 클리어 난이도가 똑같은 스테이지
- 아이템 효과가 너무 약해서 체감이 안 되는 밸런스
- 후반 스테이지에서 아이템 드롭률을 너무 낮춰 박탈감 유발

---

## 3. 돈으로 실력을 살 수 없다

> "모든 플레이어는 동일한 조건에서 시작한다"

- 가챠 보상은 **100% 코스메틱** — 볼 스킨, 이펙트 강화만
- 게임 승패에 영향을 주는 어떤 요소도 결제로 획득 불가
- 광고는 자발적 선택, 강제 노출 없음
- 재화(마법 코인)는 플레이와 광고로만 획득 가능

**이 원칙에 위배되는 것들:**
- 결제 시 라이프 무한 제공
- 결제 시 아이템 드롭률 증가
- 광고 미시청 시 스테이지 진입 불가

---

## 4. 판타지 세계관은 껍데기가 아니다

> "벽돌깨기에 RPG 옷만 입히는 게 아니라, 세계관이 게임 경험에 녹아있어야 한다"

- 아이템 이름은 마법/판타지 용어로 통일 (마법 관통석, 마법 분열구 등)
- 스테이지 테마는 숲 → 던전 → 마법사 탑 → 마왕의 방으로 서사가 있어야 함
- 효과음, BGM, 이펙트 모두 판타지 분위기에 맞게 제작
- 가챠 연출도 "뽑기"가 아닌 "마법 소환" 느낌으로

**이 원칙에 위배되는 것들:**
- 판타지 테마와 무관한 현대적 UI 디자인 사용
- 아이템을 단순히 "관통", "멀티볼"로만 표기
- 스테이지 배경이 단순 색상 변경에 그치는 것

---

## 5. 단순함이 최고의 UX다

> "설명이 필요 없는 게임을 만든다"

- 조작법은 튜토리얼 없이도 직관적으로 파악 가능해야 함
- 화면에 동시에 표시되는 정보는 최소화
- 새로운 기능 추가 시 "이게 없으면 게임이 재미없나?" 먼저 질문
- 복잡한 시스템보다 잘 다듬어진 심플한 시스템 우선

**이 원칙에 위배되는 것들:**
- HUD에 5개 이상의 정보 동시 표시
- 아이템 효과 설명이 2줄 이상 필요한 복잡한 메카닉
- 가챠 재화가 2종류 이상으로 늘어나는 것

---

## 의사결정 체크리스트

새로운 기능이나 변경사항을 추가하기 전에 아래를 확인합니다.

- [ ] 1판 플레이 시간이 3분을 초과하게 만드는가?
- [ ] 아이템의 전략적 가치를 떨어뜨리는가?
- [ ] 결제 여부가 게임 승패에 영향을 주는가?
- [ ] 판타지 세계관에서 이질감이 느껴지는가?
- [ ] 설명 없이는 이해하기 어려운 복잡함을 추가하는가?

하나라도 "예"라면 재검토합니다.
