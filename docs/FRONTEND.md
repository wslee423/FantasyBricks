# FRONTEND.md — UI/UX 구현 가이드

## 개요
Flutter 기반 UI 구현의 화면 구조, 네비게이션,
컴포넌트 패턴을 정의합니다.
DESIGN.md의 컬러/타이포 시스템을 기반으로 합니다.

---

## 화면 흐름 (Screen Flow)

```
스플래시
  ↓
로비 (메인)
  ├─ 스테이지 선택
  │    ↓
  │   게임 플레이
  │    ↓
  │   결과 화면
  │    ├─ 다음 스테이지 → 게임 플레이
  │    └─ 로비로 → 로비
  ├─ 컬렉션
  └─ 가챠
```

---

## 네비게이션 구조

### 하단 탭바 (Bottom Navigation Bar)
게임 외부 화면에서만 표시, 게임 플레이 중에는 숨김

| 탭 | 아이콘 | 화면 |
|----|--------|------|
| 🏰 모험 | 성 아이콘 | 로비 + 스테이지 선택 |
| 💎 컬렉션 | 보석 아이콘 | 보유 스킨 목록 |
| ✨ 소환 | 마법진 아이콘 | 가챠 화면 |

### 라우팅 구조
```dart
// 주요 라우트 정의
/splash          → SplashScreen
/lobby           → LobbyScreen (하단 탭 포함)
/stage-select    → StageSelectScreen
/game            → GameScreen (Flame)
/result          → ResultScreen
/collection      → CollectionScreen
/gacha           → GachaScreen
/tutorial        → TutorialOverlay (게임 위 오버레이)
```

## 화면별 상세 정의

### 1. 스플래시 화면 (SplashScreen)
- 표시 시간: 1.5초
- 구성: 게임 타이틀 + 마법진 로딩 애니메이션
- 완료 후: 튜토리얼 완료 여부 확인 → 미완료 시 튜토리얼, 완료 시 로비

---

### 2. 로비 화면 (LobbyScreen)
```
┌─────────────────────┐
│  [설정 아이콘]  상단우 │
│                     │
│   🏰 게임 타이틀     │
│                     │
│  ┌───────────────┐  │
│  │  마지막 스테이지 │  │
│  │  진행 현황 카드 │  │
│  └───────────────┘  │
│                     │
│  [▶ 모험 계속하기]   │  ← Primary 버튼
│                     │
│  💰 마법 코인: 120   │
│                     │
├─────────────────────┤
│  🏰    💎    ✨      │  ← 하단 탭바
└─────────────────────┘
```

**주요 요소:**
- 마지막 플레이 스테이지 및 별점 카드
- 보유 마법 코인 표시
- "모험 계속하기" 버튼 → 스테이지 선택으로 이동

---

### 3. 스테이지 선택 화면 (StageSelectScreen)
```
┌─────────────────────┐
│ ← 뒤로     스테이지  │
├─────────────────────┤
│  [1구간: 숲/초원]    │
│  ┌──┐ ┌──┐ ┌──┐   │
│  │1★★★│2★★☆│3☆☆☆│  │
│  └──┘ └──┘ └──┘   │
│                     │
│  [2구간: 던전/마법사탑]│
│  ┌──┐ ┌──┐ ...    │
│                     │
│  [3구간: 암흑 성채]  │
│  ┌──┐ ┌──┐ ┌──┐   │
│  │8🔒│9🔒│10🔒│     │
└─────────────────────┘
```

**주요 요소:**
- 구간별 섹션 헤더 (테마명 표시)
- 스테이지 카드: 번호 + 별점 (1~3성) + 잠금 여부
- 미클리어 스테이지는 이전 스테이지 클리어 후 해금
- 스테이지 탭 → 게임 플레이 진입

---

### 4. 게임 플레이 화면 (GameScreen)
```
┌─────────────────────┐
│ ❤️❤️❤️  [아이템HUD]  │  ← HUD 영역 (10%)
├─────────────────────┤
│                     │
│    [벽돌 영역]       │  ← 게임 영역 (75%)
│                     │
│       ● (볼)        │
│    ════════ (패들)   │
├─────────────────────┤
│   [터치 입력 영역]   │  ← 조작 영역 (15%)
└─────────────────────┘
```

**HUD 요소:**
- 좌측: 하트 아이콘 × 라이프 수
- 중앙: 활성 아이템 + 남은 타격 횟수 프로그레스 바
- 우측: 일시정지 버튼

**볼 재시작 UI:**
- 볼 이탈 후 패들 위에 볼 재생성
- 화면 중앙에 "탭하여 발사" 힌트 텍스트 표시
- 플레이어 탭 시 볼 발사

**일시정지 팝업:**
- 재개 / 처음부터 / 로비로 이동

## 컴포넌트 패턴

### 공통 컴포넌트
```dart
ArcaneButton          // Primary/Secondary 버튼
ArcaneCard            // 카드 컨테이너
StageCard             // 스테이지 선택 카드
ItemHUD               // 아이템 활성화 상태 표시
LifeIndicator         // 하트 라이프 표시
CoinDisplay           // 마법 코인 표시
StarRating            // 별점 (1~3성)
LoadingOverlay        // 로딩 오버레이
BallLaunchHint        // 볼 재시작 후 "탭하여 발사" 힌트
```

### 상태 관리
- **패키지**: `riverpod` (flutter_riverpod)
- **전역 상태**: 보유 코인, 스테이지 진행도, 장착 스킨
- **로컬 상태**: 각 화면 내 UI 상태 (탭 선택, 애니메이션 등)

## UX 원칙

| 원칙 | 구현 방법 |
|------|----------|
| 터치 영역 최소 44dp | 모든 버튼/아이콘 터치 영역 보장 |
| 한 손 조작 | 핵심 버튼은 화면 하단 배치 |
| 즉각적 피드백 | 탭 시 100ms 내 시각 반응 |
| 로딩 최소화 | 화면 전환 200ms 이내 목표 |
| 실수 방지 | 비가역적 액션 (코인 소모 등) 확인 팝업 |
